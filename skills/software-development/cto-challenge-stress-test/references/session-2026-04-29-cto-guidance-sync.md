# Session note: CTO guidance sync from external source review

Use this note when external CTO articles are mined for durable operating changes and the company CTO guide must be updated.

## Sources used
- Runn: https://www.runn.io/blog/challenges-of-being-a-new-cto
- CTO Academy: https://cto.academy/cto-responsibilities-in-start-ups-and-fast-growth-businesses

## Durable deltas extracted
- Add an inheritance audit for new CTO transitions.
- Make capacity visible by category: committed delivery, maintenance, firefighting, transformation.
- Explicitly prevent the CTO from becoming the operational critical path.
- Add a transparent yes / no / not yet prioritization rubric.
- Add change-management support for major transformations.
- Keep the source list visible in the edited guide so future reviews can trace the update.

## Practical workflow
1. Strip vendor pitches and generic leadership prose.
2. Convert source claims into short, durable checklist items.
3. Patch AGENTS.md and SOUL.md together when the same operating rule belongs in both.
4. Add a source-reference block to the guide that lists the external URLs used.
5. Verify the docs after patching to ensure the new items landed in the intended sections.
