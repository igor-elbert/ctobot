# External CTO-traits enrichment

Use this note when comparing CTO-values articles or trait lists against the governing operating manual.

## What this session reinforced
- Checklist format works best for values and operating-style updates.
- External CTO-trait sources often add two useful dimensions beyond core engineering execution:
  - cross-functional influence / executive presence
  - external credibility / market-facing communication
- When a source comparison reveals missing traits, convert them into short checklist items rather than prose paragraphs.
- Keep the operating manual and companion guide aligned when the same value or operating-style language is added.

## Practical workflow
1. Extract the external source into a small set of durable traits.
2. Compare them against the governing manual and companion guide.
3. Add only the traits that are durable and non-duplicative.
4. Prefer checklist bullets over narrative exposition.
5. Patch both AGENTS.md and SOUL.md together if the operating style is shared.
6. Verify the files after patching to avoid drift.

## Notes
- Forbes-style content may be blocked by anti-bot protection; if so, rely on accessible sources or note that the source could not be extracted reliably.
- Medium articles can be useful for the same comparison, but they should still be reduced to durable checklist items before editing the docs.
