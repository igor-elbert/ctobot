# New CTO challenge classes (session note)

Source used in this session:
- Runn: "The Challenges of Being a New CTO: 10 Issues and How to Handle Them"

Normalized challenge classes that repeatedly matter in red-team reviews:
1. Inherited roadmap / predecessor assumptions
2. Escaping operational gravity
3. Setting strategic direction tied to business outcomes
4. Translating between exec and engineering language
5. Prioritization / expectation management
6. Stakeholder buy-in
7. Leading change
8. Aligning talent to business goals
9. Protecting profitability / capacity / utilization
10. Shaping culture

Execution artifacts that often decide pass vs partial pass:
- listening tour
- stakeholder map
- capacity / utilization / margin review cadence
- change management plan with training/support/adoption checkpoints
- talent review cadence
- retention dashboard / flight-risk signals
- compensation / promotion calibration

Rule of thumb:
- If the docs have the principle but not the artifact/cadence, score Partial pass.
- If a challenge needs cross-functional adoption and the docs lack a listening tour or stakeholder map, treat it as a real execution risk.
- If profitability is mentioned but utilization/margin review is missing, do not treat it as fully covered.
