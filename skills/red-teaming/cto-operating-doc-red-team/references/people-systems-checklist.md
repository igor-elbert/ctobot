# People / systems red-team checklist

Use this checklist when stress-testing AGENTS.md / SOUL.md against external CTO challenge lists.

Recurring failure classes surfaced in this session:
- Canonical metric governance missing
  - no single source of truth for metrics
  - dashboards disagree
  - no reconciliation owner or board-day override rule
- Compliance data inventory missing
  - no data map / processing register / audit response workflow
- Performance and conflict intervention missing
  - no stay-interview / retention-risk trigger
  - no mediation path for interpersonal conflict
  - no role-reversion path for failed manager promotion
  - no knowledge-transfer sequence for performance exits
- Capacity / profitability triage missing
  - no utilization / margin cadence
  - no skeleton-crew / runway freeze plan
  - use the document's exact wording before calling this absent; close variants may still satisfy the artifact requirement
- Vendor / framework lifecycle missing
  - no deprecation trigger, migration gate, or supportability threshold
  - this is a recurring red-team gap; treat it as Partial unless a clear lifecycle rule exists
- Verification gotcha from the latest run
  - don't mark a capability as missing just because the exact keyword differs from the search phrase
  - compare the intended artifact to the actual wording and only call it a true miss if the underlying machinery is absent
- External reference handling
  - when the user asks for a source-backed comparison, cite the reference explicitly in the output instead of paraphrasing it away

Minimum artifact checks:
- named owner
- named cadence
- named escalation threshold
- named recovery / rollback / handoff plan
- named source reference when the output is based on a specific external article or PDF
